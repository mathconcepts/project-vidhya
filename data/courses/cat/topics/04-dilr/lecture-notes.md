# Data Interpretation & Logical Reasoning (DILR) — CAT

## What DILR Tests

DILR is the most polarising section in CAT. It's not testing raw intelligence — it's testing your ability to **decode structured information under time pressure** and **chain logical deductions** without making arithmetic errors.

The section has 24 questions split across 6 sets (4 questions per set). Sets come in two flavours:
- **DI (Data Interpretation)**: Tables, bar charts, pie charts, line graphs, caselets — extract numbers and calculate
- **LR (Logical Reasoning)**: Arrangements, grouping, sequencing, scheduling, puzzles, venn diagrams

## Core Concepts

### Data Interpretation

**1. Tables**
The most common DI format. Key skills:
- Read row/column headers carefully (units matter!)
- Calculate % change: `(New - Old) / Old × 100`
- Calculate % of total: `Part / Total × 100`
- Approximate aggressively — CAT DI doesn't need exact arithmetic for most questions

**2. Bar Charts & Line Graphs**
- Identify what the x-axis and y-axis measure
- For stacked bars, the segment you need is usually the difference between two cumulative values
- For line graphs, slope = rate of change; steeper = faster change

**3. Pie Charts**
- Central angle = (Part / Total) × 360°
- Percentage = central angle / 3.6
- Often combined with absolute value to find actual numbers

**4. Caselets (Paragraph DI)**
- Most calculation-heavy; often requires building your own table from the text
- Read twice, draw a table/matrix immediately
- Most students waste time re-reading — the table is your anchor

### Logical Reasoning

**1. Linear & Circular Arrangements**
- Draw a row/circle immediately; don't try to hold positions in your head
- Start with fixed positions (absolute clues before relative ones)
- "A is not adjacent to B" — mark these as negative constraints; apply them last

**2. Grouping & Selection**
- List all items, build a constraint grid
- If A → B (A selected means B selected), mark both directions
- Contradiction detection: if A → B and A → not-B, then A can never be selected

**3. Scheduling / Calendar**
- Build a day/slot matrix
- Fixed slots first, then relative constraints (X is 2 days before Y)

**4. Blood Relations**
- Draw the family tree immediately; never try to trace relations in your head
- "Cousin" is the most commonly misread relation — CAT uses both maternal and paternal

**5. Direction & Distance**
- Always orient yourself to North before starting
- Left/Right depends on the person's facing direction — draw arrows, not assumptions

**6. Set Theory / Venn Diagrams**
- 3-circle Venn: 7 regions (a, b, c, ab, bc, ca, abc, none)
- Total = sum of all regions
- Common trap: "at least 2" = ab + bc + ca + abc (not just the centre)

## CAT-Specific Strategy

### The Set Selection Trap
Not all sets are equal. CAT 2024 had sets ranging from "solvable in 4 minutes" to "25 minutes for 4 questions." The skill is **set selection**:

1. Skim ALL 6 sets in the first 2 minutes
2. Rank by: (a) familiarity with format, (b) number of constraints, (c) calculation intensity
3. Attempt your top 3 sets first; return to others if time permits

A student who perfectly solves 3 sets (12 questions) outscores a student who partially solves 6 sets.

### The 2-Question Rule
If after 4 minutes you have 0 questions solved from a set, move on. Sunk cost is real here — don't waste 12 minutes getting 0 marks.

### Approximation in DI
CAT DI options are usually spread far enough apart that approximations work:
- `37.8% ≈ 38%` is almost always safe
- Calculate to 2 significant figures, check which option is closest
- Exception: when two options are very close (e.g., 34.2% and 35.1%) — calculate precisely

### Avoiding Errors in LR
- Redraw the arrangement diagram for each question — don't carry over assumptions
- Read "exactly" vs "at least" vs "at most" very carefully
- In arrangements, if a question says "if X sits at position 3", that's a new constraint for that question only

## Time Management

| Strategy | Target |
|----------|--------|
| Set selection sweep | 2–3 min |
| Per set (3 prioritised) | 6–8 min each |
| Buffer for attempts | 5 min |
| Total | 40 min |

Target: 14–18 correct answers (out of 24) = strong 90+ percentile in DILR

## Common Mistakes

1. **Not drawing immediately**: Students who try to hold the arrangement/table in their head make 3× more errors
2. **Equal time on all sets**: DILR rewards ruthless prioritisation — abandon unfamiliar set types early
3. **Unit errors in DI**: "Figures in ₹ crore" and "Figures in ₹ thousand" in the same table = guaranteed mistake if not checked
4. **Over-inferring in LR**: Only use the constraints given. If it's not stated, it's not a constraint.
5. **Ignoring TITA questions**: No negative marking — always attempt TITA even with 30% confidence
